# Hướng dẫn Thiết lập Mac Mini M4 32GB làm Server AI với LM Studio\n\nTài liệu này hướng dẫn cách tối ưu hoá máy Mac Mini M4 (32GB Unified Memory) và thiết lập LM Studio để trở thành một AI Server nội bộ mạnh mẽ, cung cấp API cho Gateway hoặc các ứng dụng khác kết nối qua mạng cục bộ.\n\n## 1. Tối ưu macOS để làm Server AI\n\nĐể đảm bảo kết nối mạng không bị gián đoạn và máy luôn sẵn sàng phục vụ các request, bạn cần cấu hình lại hệ điều hành:\n\n### 1.1 Tắt chế độ ngủ (Prevent Sleep)\n> [!IMPORTANT]\n> Máy Mac sẽ ngắt hoặc tạm ngưng các kết nối mạng khi đi vào chế độ ngủ (Sleep Mode). Bạn bắt buộc phải tắt tính năng này để làm máy chủ chạy xuyên suốt.\n\n- Vào **System Settings > Displays > Advanced**.\n- Bật tuỳ chọn: **Prevent automatic sleeping when the display is off**.\n- *(Khuyên dùng)*: Cài ứng dụng **Amphetamine** (ứng dụng miễn phí trên App Store) để dễ dàng bật tắt chế độ giữ máy luôn thức.\n\n### 1.2 Thiết lập IP tĩnh (Static IP)\n- Vào **System Settings > Network** > Chọn mạng bạn đang sử dụng (Wi-Fi hoặc Ethernet) > **Details... > TCP/IP**.\n- Chỉnh IPv4 từ *DHCP* sang **Manually**.\n- Đặt IP theo dạng tĩnh của dải mạng nhà bạn, ví dụ: `192.168.1.100`.\n\n### 1.3 Cấu hình Tường lửa (Firewall)\n- Vào **System Settings > Network > Firewall**.\n- Nếu tường lửa đang mở, hãy bấm vào Options, đảm bảo ứng dụng **LM Studio** được cấp phép trạng thái **Incoming Connections** (Cho phép nhận kết nối đến).\n\n---\n\n## 2. Tối ưu LM Studio tận dụng sức mạnh M4\n\nKiến trúc của M4 có bộ nhớ chia sẻ (Unified Memory) tuyệt vời giữa CPU và GPU, bạn cần bật tính năng tăng tốc chuyên biệt.\n\n> [!TIP]\n> Với 32GB RAM, máy của bạn có thể host nhẹ nhàng các model chuẩn GGUF (khoản Q4_K_M hoặc Q6_K) cỡ từ Llama-3 8B, Qwen-2.5 14B đến 32B. Khuyến nghị giữ lại tối thiểu 4-6GB RAM tĩnh cho hệ điều hành macOS.\n\n### 2.1 Bật Apple Metal (GPU Acceleration)\n- Nhìn biểu mẫu bên phải ứng dụng khi Load Model (Hardware Settings).\n- Bật tuỳ chọn **Apple Metal (GPU)**.\n- Kéo thanh trượt **GPU Offload** lên mức **Max**. (Thiết lập này giúp LM Studio dồn toàn bộ lượng lớn tham số vào GPU của chip M4 thay vì CPU, gia tăng tốc độ phản hồi cực lớn).\n\n### 2.2 Cấu hình Context Length\n- Dựa trên ưu thế 32GB RAM, bạn có thể tự tin nâng **Context Length** lên ngưỡng cao như `8192` hoặc `16384` (thậm chí `32768`) nếu muốn phân tích các file code hoặc văn bản dài.\n- **Lưu ý**: Context càng dài sẽ càng ngốn RAM tĩnh. Hãy theo dõi mức tiêu thụ qua ứng dụng Activity Monitor để tìm ra con số giới hạn phù hợp nhất.\n\n---\n\n## 3. Thiết lập mạng cho "Local Server" LM Studio\n\nBước này là bước cốt lõi để biến LM Studio thành Open Gateway phục vụ tất cả các thiết bị mở rộng trong mạng LAN.\n\n> [!WARNING]\n> Mặc định LM Studio khoá Server ở mạng máy chủ vật lý (`127.0.0.1 - localhost`), ngăn chặn các máy nội bộ khác gọi tới mã lỗi 201. Bạn bắt buộc phải đổi Interface này sang `0.0.0.0`.\n\n- Mở tab hình Server (ký hiệu `<->`) trong ứng dụng LM Studio.\n- Hiện ở mục **Base network interface**, đổi giá trị từ `localhost` sang thành `0.0.0.0`.\n- Cũn tại cửa sổ cài đặt này, mở tuỳ chọn nâng cao, chọn **Enable CORS (ON)**.\n- Port mặc định giữ nguyên: `1234`.\n- Bấm **Start Server**.\n\n---\n\n## 4. Kết nối Gateway API trên máy riêng biệt\n\nTại các máy trạm hoặc cấu hình API Gateway khác của bạn, đổi Base URL từ OpenAI / localhost thành địa chỉ LAN Static MAC mà bạn đã đổi:\n\n```env\n# Cấu hình lại ứng dụng kết nối \nOPENAI_BASE_URL=\"http://192.168.1.100:1234/v1\"\n```\n\n### Kiểm tra tín hiệu bằng Command Line:\n```bash\ncurl http://192.168.1.100:1234/v1/models\n```\n*(Nếu kết quả trả về là một chuỗi mảng Data JSON mô tả thông tin các loại Model đang được LM Studio load, thì bạn đã setup thành công máy chủ AI tự hành của riêng mình!)*\n
